# 評価基準3: 成長評価（Growth Evaluation）

## 概要

成長評価は、評価期間中における社員の成長・発展を評価する軸です。「どれだけ成長したか」「どのような新しい能力を身につけたか」に焦点を当てます。

**特徴:**
- 評価者（上長）が直接評価を行います
- 最も顕著な成長カテゴリを1つ選択します
- 選択したカテゴリについて、4段階（T1〜T4）で達成度を評価します
- カテゴリごとに係数が設定されており、スコアに反映されます

---

## 成長カテゴリ

評価期間中に最も成長が見られた分野を、以下のカテゴリから1つ選択します。

### 1. 専門スキル向上（Professional Skill Development）

**定義:** 担当業務に直結する専門的なスキルの向上

**具体例:**
- プログラミング言語の新規習得
- 設計スキルの向上
- 営業手法の改善
- 分析能力の強化
- プレゼンテーション能力の向上

**係数:** 1.0（標準）

---

### 2. 資格取得・認定（Certification）

**定義:** 業務に関連する資格の取得や認定の獲得

**具体例:**
- 国家資格の取得（情報処理技術者、簿記など）
- ベンダー資格の取得（AWS、Azure、Salesforceなど）
- 業界認定の取得（PMP、CISSPなど）
- 社内認定制度の達成

**係数:** 1.1（資格取得の努力を評価）

---

### 3. 業務知識深化（Domain Knowledge）

**定義:** 担当領域における業務知識の深化・拡大

**具体例:**
- 業界動向への理解深化
- 顧客ビジネスへの理解向上
- 法規制・コンプライアンスの知識習得
- 自社製品・サービスの深い理解
- 関連分野への知識拡大

**係数:** 1.0（標準）

---

### 4. 部下・後輩指導（Mentoring）

**定義:** 後進の育成・指導における成長

**具体例:**
- 新人のOJT担当として成果を出した
- 後輩の技術指導を行い、成長を促した
- チーム内の勉強会を企画・運営した
- マニュアル・ドキュメントを整備して共有した

**係数:** 1.2（組織への波及効果を評価）

---

### 5. リーダーシップ発揮（Leadership）

**定義:** チームやプロジェクトを率いる能力の向上

**具体例:**
- プロジェクトリーダーとしてチームをまとめた
- 困難な状況でメンバーを鼓舞し、目標達成に導いた
- 部門横断の取り組みをリードした
- 意思決定力・判断力の向上が見られた

**係数:** 1.3（リーダーシップの希少性を評価）

---

### 6. 問題解決・改善提案（Problem Solving）

**定義:** 問題解決能力や改善提案力の向上

**具体例:**
- 長年の課題を解決した
- 業務プロセスの大幅な改善を実現した
- トラブル対応力が向上した
- 論理的思考力の向上が見られた

**係数:** 1.1（実務への貢献を評価）

---

### 7. 新規領域への挑戦（New Challenge）

**定義:** 未経験の領域への挑戦と適応

**具体例:**
- 新しい技術領域へのチャレンジ
- 異なる職種・役割への適応
- 新規事業・プロジェクトへの参画
- 海外業務への対応

**係数:** 1.2（挑戦する姿勢を評価）

---

## 達成度レベル（T1〜T4）

選択したカテゴリについて、以下の4段階で達成度を評価します。

| レベル | 説明 | スコア |
|--------|------|--------|
| **T4（卓越）** | 期待を大きく上回る成長。組織に顕著な貢献 | 5.0 |
| **T3（良好）** | 期待通りの成長。着実な進歩が見られる | 3.5 |
| **T2（普通）** | 一定の成長は見られるが、期待には届かない | 2.5 |
| **T1（不十分）** | 成長が限定的。より積極的な取り組みが必要 | 1.0 |

### 各レベルの判断基準

**T4（卓越）を選ぶ場合:**
- 短期間で顕著な成長を遂げた
- 習得した能力を即座に業務で発揮し、成果を出した
- 周囲にも良い影響を与えた
- 期待を大きく超える達成

**T3（良好）を選ぶ場合:**
- 計画通りの成長を遂げた
- 習得した能力を業務で活用できている
- 着実な進歩が確認できる
- 期待通りの達成

**T2（普通）を選ぶ場合:**
- 成長の意欲は見られるが、結果が伴わない
- 習得途中で、まだ業務に活かせていない
- 成長のペースが遅い
- 期待にはやや届かない

**T1（不十分）を選ぶ場合:**
- 成長への取り組みが見られない
- 学習の機会を活かせていない
- 同じ失敗を繰り返している
- 成長意欲が感じられない

## スコア計算ロジック

### 基本計算式

```
成長評価スコア = レベルスコア × カテゴリ係数
```

ただし、上限は 5.0 とします。

### 計算例

**例1: リーダーシップ発揮（係数1.3）でT4（スコア5.0）の場合**
```
成長評価スコア = 5.0 × 1.3 = 6.5 → 5.0（上限適用）
```

**例2: 部下・後輩指導（係数1.2）でT3（スコア3.5）の場合**
```
成長評価スコア = 3.5 × 1.2 = 4.2
```

**例3: 専門スキル向上（係数1.0）でT2（スコア2.5）の場合**
```
成長評価スコア = 2.5 × 1.0 = 2.5
```

### 係数の意味

係数は、各カテゴリの「組織への貢献度」や「難易度」を反映しています：

| 係数 | 意味 | 該当カテゴリ |
|------|------|-------------|
| 1.0 | 標準的な成長 | 専門スキル向上、業務知識深化 |
| 1.1 | やや高い貢献 | 資格取得・認定、問題解決・改善提案 |
| 1.2 | 高い貢献 | 部下・後輩指導、新規領域への挑戦 |
| 1.3 | 非常に高い貢献 | リーダーシップ発揮 |

---

## 重み付け

成長評価のスコアは、資格等級に応じた重みで最終スコアに反映されます。

| 資格等級 | 成長評価の重み | 説明 |
|----------|---------------|------|
| G1-G2（一般） | 40% | 成長重視（育成期） |
| G3-G4（主任級） | 30% | バランス型 |
| G5-G6（管理職） | 25% | 成果・プロセス重視 |
| G7以上（上級管理職） | 20% | 成果最重視 |

**例:** 成長評価スコア 4.2、重み 30% の場合
```
加重スコア = 4.2 × 0.30 = 1.26
```

## カテゴリ選択のガイドライン

### 選択の基本原則

1. **最も顕著な成長を選ぶ**
   - 複数の成長があっても、最も際立ったものを1つ選択
   - 「成長の幅」と「成長の深さ」の両面から判断

2. **評価期間内の成長を対象とする**
   - 過去から持っている能力ではなく、期間中に伸びた能力
   - 「以前と比べてどう変わったか」が重要

3. **業務との関連性を考慮**
   - 趣味的な学習より、業務に活かせる成長を優先
   - ただし、間接的に業務に貢献するものも評価対象

### カテゴリ選択のフローチャート

```
Q1: 新しい資格を取得したか？
  → Yes: 「資格取得・認定」を検討
  → No: Q2へ

Q2: 後輩・チームメンバーの指導で成果があったか？
  → Yes: 「部下・後輩指導」を検討
  → No: Q3へ

Q3: プロジェクトやチームをリードしたか？
  → Yes: 「リーダーシップ発揮」を検討
  → No: Q4へ

Q4: 未経験の領域に挑戦したか？
  → Yes: 「新規領域への挑戦」を検討
  → No: Q5へ

Q5: 業務改善や問題解決で成果があったか？
  → Yes: 「問題解決・改善提案」を検討
  → No: Q6へ

Q6: 専門スキルが向上したか？
  → Yes: 「専門スキル向上」を検討
  → No: 「業務知識深化」を検討
```

## よくある質問

### Q1: カテゴリが決められない場合はどうすればいいですか？

**A:** 以下の方法を試してください：
1. 被評価者本人に「今期最も成長したと思う点」を聞く
2. 具体的な成果やエピソードを洗い出し、どのカテゴリに該当するか検討
3. 複数該当する場合は、より「組織への貢献度」が高いものを選ぶ

### Q2: 成長が見られない場合はどう評価しますか？

**A:** 成長が限定的な場合でも、何らかのカテゴリを選択し、T1またはT2で評価します。完全に成長がゼロということは稀です。小さな成長でも見つけ出し、評価することが重要です。

### Q3: 係数の高いカテゴリを選んだ方が有利ですか？

**A:** 係数が高くても、達成度が低ければスコアは低くなります。
- リーダーシップ（係数1.3）× T2（2.5）= 3.25
- 専門スキル（係数1.0）× T4（5.0）= 5.0

実態に合ったカテゴリと達成度を選ぶことが重要です。

### Q4: 同じカテゴリを毎期選んでもいいですか？

**A:** 問題ありません。継続的に成長している分野であれば、同じカテゴリで評価するのは自然です。ただし、毎期T4をつける場合は、本当に「卓越した成長」があったかを慎重に判断してください。

### Q5: 成長評価のコメントには何を書けばいいですか？

**A:** 以下の要素を含めると効果的です：
1. **選択したカテゴリの理由**
2. **成長を示す具体的なエピソード**
3. **成長がもたらした業務への効果**
4. **今後さらに伸ばしてほしい点**

## コメント記載例

### 良いコメント例

**カテゴリ: リーダーシップ発揮 / 達成度: T4**
```
今期、新規プロジェクトAのリーダーに抜擢され、5名のチームを
率いて目標を達成した。特に、メンバー間の役割分担を明確にし、
進捗管理を徹底することで、予定より2週間早くリリースできた。
また、チーム内で困っているメンバーには積極的に声をかけ、
心理的安全性の高いチーム環境を構築した。
リーダーシップの成長は目覚ましく、今後の活躍に期待する。
```

**カテゴリ: 専門スキル向上 / 達成度: T3**
```
今期、Pythonによるデータ分析スキルを習得し、月次レポートの
自動化を実現した。独学で基礎を学んだ後、実務で活用しながら
スキルを磨き、現在ではチーム内でPythonに関する相談を
受けるまでになった。着実な成長が見られる。
次期は、機械学習の基礎にも挑戦することを期待する。
```

**カテゴリ: 部下・後輩指導 / 達成度: T3**
```
今期、新入社員2名のOJTを担当し、3ヶ月で独り立ちできるまで
育成した。週1回の1on1を欠かさず実施し、技術面だけでなく
メンタル面のサポートも行った。新入社員からの信頼も厚く、
指導者としての成長が見られた。
今後は、より体系的な育成プランの策定にも取り組んでほしい。
```

## 評価者へのアドバイス

### 成長を見つける視点

成長評価は「成長を見つける」評価です。以下の視点で観察してください：

1. **Before / After の比較**
   - 期初と期末で何が変わったか
   - できなかったことができるようになったか

2. **チャレンジの有無**
   - 新しいことに挑戦したか
   - コンフォートゾーンを超えた行動があったか

3. **失敗からの学び**
   - 失敗した場合、そこから何を学んだか
   - 同じ失敗を繰り返さなくなったか

4. **周囲への影響**
   - 成長が周囲に良い影響を与えたか
   - ロールモデルになっているか

### 成長を促す評価面談

1. **成長を承認する**
   - まず、成長した点を具体的に伝える
   - 努力のプロセスも認める

2. **次の成長目標を共有する**
   - 今後伸ばしてほしい能力を伝える
   - 本人のキャリア志向も聞く

3. **成長を支援する姿勢を示す**
   - 必要なリソースや機会を提供する約束
   - 定期的なフィードバックの継続

成長評価は、被評価者の「伸びしろ」を評価する軸です。過去の実績だけでなく、未来への可能性を見出し、成長を後押しする評価を心がけてください。
