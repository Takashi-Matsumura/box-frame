# 目標設定ガイド（Goal Setting Guide）

## はじめに

目標設定は、評価期間の始まりに行う重要なステップです。適切な目標を設定することで、日々の業務に方向性が生まれ、評価期間終了時に自分の成長を振り返りやすくなります。

---

## 目標設定の基本的な考え方

### なぜ目標を設定するのか

目標設定は「評価のため」だけではありません。以下のような効果があります：

- **方向性の明確化**: 何に注力すべきかが明確になる
- **成長の可視化**: 期間終了後に自分の成長を客観的に振り返れる
- **上長との共通認識**: 期待される役割や成果について認識を合わせられる
- **モチベーション向上**: 達成感を得やすくなる

### 「良い目標」の条件（SMART 原則）

効果的な目標は、以下の条件を満たしています：

| 要素                       | 説明                         | 例                          |
| -------------------------- | ---------------------------- | --------------------------- |
| **S**pecific（具体的）     | 曖昧でなく、何をするか明確   | × 頑張る → ○ 〇〇を完了する |
| **M**easurable（測定可能） | 達成したかどうか判断できる   | × 改善する → ○ 20%削減する  |
| **A**chievable（達成可能） | 努力すれば達成できる範囲     | 現実的で、かつ挑戦的        |
| **R**elevant（関連性）     | 業務や組織目標に関連している | 自分の役割と一致している    |
| **T**ime-bound（期限）     | いつまでに達成するか明確     | 評価期間内に達成可能        |

## プロセス目標の設定

### プロセス目標とは

プロセス目標は、「どのような仕事の進め方・取り組み方をするか」に焦点を当てた目標です。日々の業務プロセスや仕事の質に関する目標を設定します。

### プロセス 1: 通常業務

すべての社員に共通する「通常業務」での目標です。日常業務における改善や工夫を目標として設定します。

**社員向けの目標例:**

1. **業務効率化**

   - 「定型業務の作業時間を 10%短縮する」
   - 「チェックリストを作成し、ミスを半減させる」
   - 「よくある質問を FAQ 化し、問い合わせ対応を効率化する」

2. **品質向上**

   - 「提出物のセルフチェックを徹底し、差し戻しをゼロにする」
   - 「ドキュメントのテンプレートを整備し、品質を均一化する」
   - 「レビュー指摘事項を記録し、同じ指摘を受けないようにする」

3. **コミュニケーション**

   - 「報告・連絡・相談を迅速に行い、タスクの滞留をなくす」
   - 「会議の議事録を当日中に共有する習慣をつける」
   - 「不明点はその日のうちに確認し、認識齟齬を防ぐ」

4. **スケジュール管理**
   - 「すべてのタスクに期限を設定し、遅延をゼロにする」
   - 「週次で進捗を上長に報告する習慣をつける」
   - 「バッファを含めた現実的なスケジュールを立てる」

### プロセス 2 以降: 特別プロジェクト

通常業務以外に、特別なプロジェクトや取り組みがある場合に追加します。

**追加プロセスの例:**

- 「新システム導入プロジェクト: テスト工程を予定通り完了させる」
- 「採用活動: 説明会で学生 5 名以上と面談する」
- 「社内勉強会: 四半期に 1 回、チーム向けに技術共有を行う」

### プロセス目標の書き方のコツ

1. **「〜する」で終わる能動的な表現を使う**

   - × 品質を意識する → ○ 品質チェックシートを毎回活用する

2. **数値や頻度を入れる**

   - × 定期的に報告する → ○ 週 1 回の進捗報告を欠かさず行う

3. **自分でコントロールできることを書く**
   - × 売上を達成する → ○ 顧客訪問を月 20 件実施する

## 成長目標の設定

### 成長目標とは

成長目標は、「評価期間中にどのような成長を遂げたいか」を設定するものです。成長カテゴリから 1 つを選び、具体的な目標を記述します。

### 成長カテゴリの選び方

**社員におすすめのカテゴリ:**

1. **専門スキル向上**（最も選びやすい）

   - 技術力や業務スキルを伸ばしたい場合
   - 例: プログラミング、設計、分析、プレゼンなど

2. **業務知識深化**

   - 担当領域の知識を深めたい場合
   - 例: 業界知識、顧客理解、製品知識など

3. **資格取得・認定**

   - 具体的な資格取得を目指す場合
   - 例: 情報処理、簿記、TOEIC、ベンダー資格など

4. **問題解決・改善提案**
   - 課題発見・解決能力を伸ばしたい場合
   - 例: 業務改善、トラブル対応力向上など

### 社員向けの成長目標例

**専門スキル向上の場合:**

- 「Python を独学で学び、簡単なデータ分析ができるようになる」
- 「Excel の VBA を習得し、定型業務を自動化する」
- 「プレゼン研修を受講し、四半期報告会で発表する」

**業務知識深化の場合:**

- 「担当顧客の業界動向を調査し、レポートにまとめる」
- 「社内製品の勉強会に参加し、全製品ラインナップを説明できるようになる」
- 「関連法規を学び、コンプライアンスチェックができるようになる」

**資格取得・認定の場合:**

- 「基本情報技術者試験に合格する」
- 「TOEIC 700 点を取得する」
- 「AWS Cloud Practitioner 認定を取得する」

**問題解決・改善提案の場合:**

- 「チーム内の課題を 1 つ特定し、改善案を提案・実行する」
- 「過去のトラブル事例を分析し、再発防止策をまとめる」
- 「業務フローの無駄を見つけ、効率化を提案する」

### 成長目標の書き方のコツ

1. **期間内に達成可能なレベルに設定する**

   - × 英語をマスターする → ○ ビジネスメールを英語で書けるようになる

2. **「できるようになる」「〜を身につける」などの表現を使う**

   - 成長を示す表現にする

3. **具体的な行動や成果物を含める**
   - × スキルを向上させる → ○ 〇〇の勉強会で発表する

## 自己評価の入力

### 自己評価とは

自己評価は、設定した目標に対して「自分がどの程度達成できたか」を振り返り、星評価（5段階）とコメントで記録するものです。評価期間の終盤に入力し、評価者との面談前に提出します。

### 星評価の基準

自己評価は5段階の星で表現します。これは評価者による評価（T1〜T4）とは異なり、**自分自身の達成感**を表すものです。

| 星 | ラベル | 意味 |
| --- | --- | --- |
| ★☆☆☆☆ | 達成できなかった | 目標にほとんど取り組めなかった、または成果が出なかった |
| ★★☆☆☆ | あまり達成できなかった | 取り組んだが、期待した成果には届かなかった |
| ★★★☆☆ | ある程度達成できた | 目標の半分程度、または主要な部分は達成できた |
| ★★★★☆ | 十分達成できた | 目標をおおむね達成でき、満足できる成果を出せた |
| ★★★★★ | 大きく達成できた | 目標を上回る成果を出せた、または大きな成長を実感できた |

### 自己評価コメントの書き方

星評価だけでなく、**コメント欄に理由や根拠を記入**することが重要です。コメントがあることで：

- 評価者があなたの取り組みを理解しやすくなる
- 面談での振り返りがスムーズになる
- 自分自身の成長を言語化できる

**コメントに含めるべき内容:**

1. **具体的な成果や取り組み**
   - 「〇〇を完了し、△△の効果があった」
   - 「月XX件の目標に対し、YY件を達成した」

2. **うまくいった点・工夫した点**
   - 「〇〇を意識して取り組んだ結果、△△できた」
   - 「困難があったが、□□することで乗り越えた」

3. **課題や反省点**（正直に書く）
   - 「〇〇については時間が足りず、十分に取り組めなかった」
   - 「△△の部分は想定より難しく、次期の課題としたい」

4. **次につなげたいこと**
   - 「この経験を活かして、次期は〇〇に挑戦したい」

### 自己評価コメントの記述例

**プロセス目標の自己評価例:**

```
★★★★☆（十分達成できた）

週次の進捗報告を欠かさず行い、タスクの滞留をゼロにできました。
特に月末の繁忙期でも報告を継続できたことは自信になりました。
ただし、報告内容の質についてはまだ改善の余地があり、
次期は「報告の分かりやすさ」も意識していきたいです。
```

**成長目標の自己評価例:**

```
★★★☆☆（ある程度達成できた）

Excel VBAの基礎を習得し、3つの定型業務を自動化できました。
作業時間は合計で週2時間程度削減できています。
ただし、当初目標としていた5つの自動化には届かず、
残り2つは次期に持ち越しとなりました。
VBAの学習は継続し、より複雑な処理にも挑戦したいです。
```

### 自己評価と評価者評価の違い

| 項目 | 自己評価 | 評価者評価 |
| --- | --- | --- |
| **評価者** | 自分自身 | 上長（評価者） |
| **評価基準** | 達成感・手応え | 期待値との比較 |
| **スケール** | 星5段階 | T1〜T4の4段階 |
| **目的** | 自己認識の言語化 | 客観的な評価 |

自己評価と評価者評価に**差異があっても問題ありません**。むしろ、その差異について面談で話し合うことが重要です。差異から新たな気づきが生まれることもあります。

## カレンダー機能の使い方

### カレンダーとは

目標設定画面には、評価期間を表示するカレンダーがあります。このカレンダーでは以下のマークを記録できます：

| マーク | 色 | 説明 |
| --- | --- | --- |
| 面談 | 青 | 評価者との面談を実施した日 |
| 自己評価完了 | 緑 | 自己評価を提出した日 |

### 面談日の記録方法

1. カレンダーの日付をクリック
2. ダイアログが表示される
3. 面談のメモ（任意）を入力
4. 「保存」をクリック

**メモの例:**
- 「目標設定面談」
- 「中間進捗確認」
- 「期末評価面談」

### 自己評価の提出

1. すべてのプロセス目標と成長目標に星評価を入力
2. コメント欄に理由・根拠を記入（推奨）
3. 「自己評価を提出」ボタンをクリック
4. 確認ダイアログで「OK」を選択

**注意:** 提出後は自己評価の編集ができなくなります。提出前に内容を十分確認してください。

## 評価者との面談に向けて

### 面談の目的

目標設定後の面談は、以下の目的で行われます：

1. **目標の共有**: 設定した目標を上長と共有し、認識を合わせる
2. **フィードバック**: 上長から目標に対するアドバイスをもらう
3. **期待の確認**: 上長が期待する役割や成果を確認する
4. **サポートの相談**: 必要な支援やリソースについて相談する

期末の面談では、さらに以下が加わります：

5. **自己評価の共有**: 自分の振り返りを伝える
6. **差異の確認**: 自己評価と評価者評価の違いについて話し合う

### 面談前の準備

1. **目標を説明できるようにしておく**

   - なぜその目標を設定したのか
   - どのように達成するつもりか
   - 何がアウトプットになるか

2. **聞きたいことをリストアップする**

   - 目標の方向性は合っているか
   - 追加で期待されていることはあるか
   - 参考になる事例やアドバイスはあるか

3. **自己評価の根拠を整理する**（期末面談の場合）
   - 具体的な成果や数字
   - 工夫した点や乗り越えた困難
   - 課題や反省点

### 面談での伝え方

1. **結論から先に話す**

   - 「今期の目標は〇〇です。理由は〜」
   - 「自己評価は★4つです。理由は〜」

2. **数字や具体例を交えて話す**

   - 抽象的な話は避け、具体的に説明する

3. **前向きな姿勢を見せる**

   - 課題を認識しつつも、前向きに取り組む姿勢を示す

4. **質問やアドバイスを積極的に求める**
   - 「この目標の方向性はいかがでしょうか？」
   - 「アドバイスがあればいただきたいです」

## よくある悩みと解決策

### Q1: 目標が思いつかない

**A:** 以下の観点から考えてみましょう：

- 前期の反省点は何か？
- 上長や先輩から指摘されたことは何か？
- 日々の業務で「もっとこうしたい」と思うことは何か？
- チームや組織の課題で、自分が貢献できることは何か？

### Q2: 目標が大きすぎる/小さすぎる気がする

**A:** 「半年後の自分」をイメージしてみましょう。

- 大きすぎる場合: 最初のステップに分解する
- 小さすぎる場合: もう一歩先の成果を考える
- 迷ったら上長に相談するのがベストです

### Q3: 達成できるか不安

**A:** 不安を感じるのは、適度に挑戦的な目標を設定できている証拠です。

- 途中経過を定期的に確認する仕組みを作る
- 困ったら早めに上長に相談する
- 100%達成できなくても、取り組みの過程が評価されます

### Q4: 他の人と比べて目標のレベルが低い気がする

**A:** 目標は他人と比較するものではありません。

- 自分の現状からどれだけ成長するかが重要
- 経験年数や役割によって適切な目標は異なる
- 自分にとって挑戦的であれば、それが良い目標です

### Q5: 業務が忙しくて目標に取り組む時間がない

**A:** 業務と目標を分けて考えないようにしましょう。

- 通常業務の中で達成できる目標を設定する
- 「効率化」の目標であれば、忙しさの解消にもつながる
- 小さな改善を積み重ねることも立派な目標達成です

### Q6: 自己評価で何を書けばいいかわからない

**A:** 以下の順番で考えてみましょう：

1. まず星の数を決める（直感でOK）
2. なぜその星の数にしたのか、理由を書き出す
3. 具体的なエピソードや数字を思い出す
4. 次に活かしたいことを一言添える

完璧な文章である必要はありません。自分の言葉で率直に書きましょう。

### Q7: 自己評価と評価者評価が違ったらどうなる？

**A:** 差異があること自体は問題ではありません。

- 差異は面談で話し合うきっかけになります
- 自分では気づかなかった強みや課題が見つかることもあります
- 大切なのは「なぜ差異があるのか」を理解することです

## まとめ

目標設定から自己評価までの流れは以下の通りです：

### 1. 目標設定（期初）
- プロセス目標と成長目標を設定
- 評価者との面談で認識を合わせる

### 2. 目標への取り組み（期中）
- 日々の業務の中で目標を意識して行動
- 必要に応じて面談で進捗を確認

### 3. 自己評価の入力（期末）
- 各目標に対して星評価を入力
- コメント欄に理由・根拠を記入
- 「自己評価を提出」で確定

### 4. 期末面談
- 自己評価を基に振り返りを共有
- 評価者からフィードバックを受ける

---

目標設定と自己評価は「評価されるため」ではなく「自分の成長を加速させるため」に行うものです。この機会を活用して、充実した評価期間を過ごしましょう。

### 目標設定チェックリスト

設定した目標を以下の観点でチェックしてみましょう：

- [ ] 具体的に何をするかが明確になっているか
- [ ] 達成したかどうか判断できる基準があるか
- [ ] 評価期間内に達成可能なレベルか
- [ ] 自分の役割や業務に関連しているか
- [ ] 自分でコントロールできる内容か
- [ ] 挑戦的だが、現実的な目標か
- [ ] 上長に説明できる内容か

### 自己評価チェックリスト

提出前に以下を確認しましょう：

- [ ] すべてのプロセス目標に星評価を入力したか
- [ ] 成長目標に星評価を入力したか
- [ ] 各評価にコメント（理由・根拠）を記入したか
- [ ] 具体的な成果や数字を含めているか
- [ ] 課題や反省点も正直に記載したか
- [ ] 提出後は編集できないことを理解しているか
