# 評価基準2: プロセス評価（Process Evaluation）

## 概要

プロセス評価は、日常業務における行動・取り組み姿勢を評価する軸です。「何を達成したか」ではなく「どのように取り組んだか」に焦点を当てます。

**特徴:**
- 評価者（上長）が直接評価を行います
- 5つの行動特性について、4段階（T1〜T4）で評価します
- 主観的な評価ですが、明確な基準に基づいて判断します

---

## 評価項目（5つの行動特性）

### 1. 主体性（Initiative）

**定義:** 自ら考え、積極的に行動する姿勢

| レベル | 説明 |
|--------|------|
| **T4（優秀）** | 課題を先読みし、自発的に解決策を提案・実行。周囲を巻き込んで推進 |
| **T3（標準）** | 指示がなくても必要な行動を自ら判断し実行できる |
| **T2（要改善）** | 指示があれば行動するが、自発的な提案は少ない |
| **T1（不十分）** | 指示待ちの姿勢が強く、受動的な行動が目立つ |

**評価のポイント:**
- 新しい取り組みへの提案があったか
- 問題発見時に自ら動いたか
- 待ちの姿勢ではなく、先回りした行動があったか

---

### 2. 協調性（Teamwork）

**定義:** チーム内での協力と円滑なコミュニケーション

| レベル | 説明 |
|--------|------|
| **T4（優秀）** | チームの潤滑油として機能。他者をサポートし、全体最適を意識した行動 |
| **T3（標準）** | チームメンバーと良好な関係を築き、必要に応じて協力できる |
| **T2（要改善）** | 協力はするが、積極性に欠ける。時に個人プレーが目立つ |
| **T1（不十分）** | チームワークを乱す行動がある。協力姿勢が見られない |

**評価のポイント:**
- 他部署・他チームとの連携状況
- 困っているメンバーへのサポート
- 情報共有の積極性
- チーム全体の成果への貢献意識

---

### 3. 責任感（Accountability）

**定義:** 与えられた役割を最後までやり遂げる意識

| レベル | 説明 |
|--------|------|
| **T4（優秀）** | 困難な状況でも諦めず、最後まで責任を持って完遂。期待以上の成果 |
| **T3（標準）** | 担当業務を確実に遂行。納期・品質を守る |
| **T2（要改善）** | 基本的な業務は遂行するが、難しい課題では逃げ腰になることがある |
| **T1（不十分）** | 業務の放棄や納期遅延が目立つ。責任転嫁の傾向がある |

**評価のポイント:**
- 納期遵守の状況
- 品質へのこだわり
- 問題発生時の対応姿勢
- 最後まで投げ出さない姿勢

---

### 4. 改善力（Continuous Improvement）

**定義:** より良い方法を模索し、業務改善に取り組む姿勢

| レベル | 説明 |
|--------|------|
| **T4（優秀）** | 業務プロセスの抜本的な改善を提案・実行。組織全体に波及する効果 |
| **T3（標準）** | 日常業務の中で改善点を見つけ、実際に改善を行う |
| **T2（要改善）** | 改善の必要性は認識しているが、具体的な行動が伴わない |
| **T1（不十分）** | 現状維持志向が強く、改善への意識が低い |

**評価のポイント:**
- 業務効率化の提案・実行
- 「なぜ」を問う姿勢
- 失敗からの学習と改善
- 新しい方法への挑戦

---

### 5. 専門性（Expertise）

**定義:** 業務に必要な知識・スキルの保有と活用

| レベル | 説明 |
|--------|------|
| **T4（優秀）** | 高度な専門知識を持ち、他者の指導や難問解決に貢献 |
| **T3（標準）** | 業務遂行に必要な知識・スキルを十分に保有し、活用できる |
| **T2（要改善）** | 基本的な知識はあるが、応用力や深さに課題がある |
| **T1（不十分）** | 業務に必要な基礎知識・スキルが不足している |

**評価のポイント:**
- 担当業務への理解度
- 専門知識の深さと幅
- 知識を実務に活かす能力
- 学習意欲と自己研鑽

---

## スコア計算ロジック

### レベルからスコアへの変換

各レベル（T1〜T4）は、以下のスコアに変換されます：

| レベル | スコア | 説明 |
|--------|--------|------|
| T1 | 1.0 | 不十分 |
| T2 | 2.5 | 要改善 |
| T3 | 3.5 | 標準 |
| T4 | 5.0 | 優秀 |

### プロセス評価スコアの算出

5つの行動特性のスコア平均値がプロセス評価スコアになります。

**計算式:**
```
プロセス評価スコア = (主体性 + 協調性 + 責任感 + 改善力 + 専門性) ÷ 5
```

**計算例:**
```
主体性: T3 = 3.5
協調性: T4 = 5.0
責任感: T3 = 3.5
改善力: T2 = 2.5
専門性: T3 = 3.5

プロセス評価スコア = (3.5 + 5.0 + 3.5 + 2.5 + 3.5) ÷ 5 = 3.6
```

---

## 重み付け

プロセス評価のスコアは、資格等級に応じた重みで最終スコアに反映されます。

| 資格等級 | プロセス評価の重み | 説明 |
|----------|-------------------|------|
| G1-G2（一般） | 40% | 行動習慣の形成重視 |
| G3-G4（主任級） | 40% | バランス型 |
| G5-G6（管理職） | 35% | 成果との両立 |
| G7以上（上級管理職） | 30% | 成果重視へシフト |

**例:** プロセス評価スコア 3.6、重み 40% の場合
```
加重スコア = 3.6 × 0.40 = 1.44
```

## 評価のガイドライン

### 評価の基本原則

1. **具体的な行動に基づく評価**
   - 印象や感覚ではなく、観察した行動事実に基づいて評価
   - 「〇〇という場面で△△という行動をとった」と説明できるように

2. **評価期間全体を通じた評価**
   - 直近の出来事だけでなく、期間全体の行動を振り返る
   - 成長や変化も考慮に入れる

3. **公平性の確保**
   - 同じ基準をすべての被評価者に適用
   - 個人的な好き嫌いを排除

4. **T3を基準とする**
   - T3（標準）は「期待される水準を満たしている」状態
   - T4は「期待を上回る優れた行動」がある場合のみ
   - 全員にT4をつけることは避ける

### 評価の偏りを防ぐ

| 偏りの種類 | 説明 | 対策 |
|-----------|------|------|
| **中心化傾向** | すべてT3に集中 | 具体的な行動差を見つける |
| **寛大化傾向** | 高評価に偏る | T4の基準を厳格に適用 |
| **厳格化傾向** | 低評価に偏る | 良い点も積極的に評価 |
| **ハロー効果** | 一つの印象が全体に影響 | 項目ごとに独立して評価 |
| **近時効果** | 直近の出来事に影響される | 期間全体を振り返る |

## よくある質問

### Q1: T3とT4の違いがわかりにくいです

**A:** T3は「期待通りの行動ができている」状態、T4は「期待を超える卓越した行動がある」状態です。

**T3の例（協調性）:**
- チームミーティングで自分の意見を述べ、他者の意見も尊重する
- 依頼された協力には応じる

**T4の例（協調性）:**
- 自発的に他部署との橋渡し役を担う
- チーム内の対立を調整し、合意形成をリード
- 困っているメンバーを自ら見つけてサポート

### Q2: 5項目すべてを同じレベルにしてもいいですか？

**A:** 結果的に同じになることはあり得ますが、各項目は独立して評価してください。人には得意・不得意があるのが自然です。すべて同じレベルの場合は、本当に各項目を個別に評価したか再確認してください。

### Q3: 評価期間中に成長した場合、どう評価しますか？

**A:** 期末時点での水準を基本としつつ、成長の過程も加味します。大きな成長が見られた場合は、コメント欄でその旨を記載し、評価に反映させることができます。

### Q4: 業務内容によって評価しにくい項目があります

**A:** 職種や役割によって、行動特性の発揮機会に差があることは事実です。その場合：
- 機会が少ない中でも発揮された行動を評価
- 類似の行動特性として評価できる場面を探す
- どうしても評価困難な場合は、T3（標準）として、コメントで補足

### Q5: プロセス評価のコメントには何を書けばいいですか？

**A:** 以下の要素を含めると効果的です：
1. **評価の根拠となる具体的な行動事例**
2. **特に優れていた点（T4をつけた項目）**
3. **改善が期待される点（T1・T2をつけた項目）**
4. **今後への期待・アドバイス**

## コメント記載例

### 良いコメント例

```
【主体性】
新規顧客開拓において、従来の手法に加えてSNSを活用した
アプローチを自ら提案・実行し、3件の商談創出につなげた。
課題を先読みした行動として高く評価できる。

【協調性】
プロジェクトXでは、開発チームと営業チームの間で認識の
ずれが生じた際、自ら調整役を買って出て、両者の合意形成を
リードした。チーム間連携の模範となる行動だった。

【改善力】
週次レポートの作成プロセスを見直し、テンプレート化と
自動集計の仕組みを構築。作業時間を50%削減した。
継続的な改善意識が高い。

【今後への期待】
専門性については基礎は十分だが、より高度な分析スキルの
習得を期待する。社内研修への参加を推奨したい。
```

### 改善が必要なコメント例

```
× 「頑張っていた」 → 具体性がない
○ 「〇〇プロジェクトで△△を達成した」

× 「もう少し積極的に」 → 曖昧
○ 「会議での発言を増やし、アイデア提案を期待」

× 「普通」 → 評価の根拠がない
○ 「担当業務を確実に遂行し、納期遵守率100%」
```

## 評価者へのアドバイス

### 日頃からの観察が重要

プロセス評価は日常の行動を評価するため、評価期間中の観察記録が重要です：

- 1on1ミーティングでの気づきをメモ
- プロジェクトでの行動をチェック
- 他メンバーからのフィードバックを収集
- 評価期間の最初・中間・最後で変化を確認

### 評価面談での伝え方

1. **良い点から伝える**（T4・T3の項目）
2. **具体的な行動事例を示す**
3. **改善点は建設的に伝える**（T1・T2の項目）
4. **今後の期待を明確にする**
5. **本人の意見も聞く**

プロセス評価は、被評価者の成長を促すための重要なフィードバック機会です。単なる点数付けではなく、対話を通じた育成を心がけてください。
